Programmation Orientee Objet



Introduction :

La Programmation Orientee Objet(P.O.O) peut se definir comme une paradigme de Programmation qui repose sur le concept de classe et d'objet. Le mot "orientee-objet" est la combinaison de deux termes, objet et orientee. Dans le dictionnaire, objet signifie quelque chose qui existe dans la vie reel et oriente signifie, focalise sur un type particulier de chose ou d'entite(objet). Dans les prochaines lignes nous parlerons de la structure de la P.O.O , ensuite des pilliers de la P.O.O.




Structure de la Programmation Orientee Objet :

Dans une structure de code basee sur la P.O.O comporte de nombreuses parties, donc programmer a chaque fois la meme serais fastidieux. Pour repondre a ce probleme nous utilisons ce qu'on appelle des classes, objets, methodes et attributs.


  1. LES CLASSES :

    2. Les classes sont des types de données définis par l'utilisateur qui servent de schéma directeur pour les objets, les attributs et les méthodes, c'est a dire qu'une classe est un ensemble de code contenant des variables et des fonctions permettant de créer des objets. Une classe peut contenir plusieurs objets.

  2. LES OBJETS :

    3. Un objet est un bloc de code mêlant des variables et des fonctions, appelées respectivement attributs et méthodes.

  3. LES ATTRIBUTS :

    4. Les attributs définissent les caractéristiques d’un objet d’une classe. Les objets contiennent des données stockees dans le champ d'un attribut.

  4. LES METHODES :

    5. Ce sont des fonctions qui sont definies dans une classe, qui decrivent le comportement d'un objet. Ils sont utiles pour leur reutilisabilite ou gardent une fonctionnalite encapsule dans un objet pendant un moment. La reutilisabilite d'un code est tres benefique lorsque nous deboguons notre code.




Les principes de la P.O.O. :

Afin de savoir comment écrire un bon code POO, nous devons comprendre les 4 piliers de la POO auxquels nous devons adhérer : Encapsulation Abstraction Héritage Polymorphisme Plongeons dans le vif du sujet et comprenons mieux ce que chacun de ces piliers signifie exactement.

  1. Encapsulation

    2. C'est le concept qui lie les données entre elles. Les fonctions manipulent l'information et la gardent en sécurité. Aucun accès direct n'est accordé à l'info au cas où elle serait cachée. Si vous souhaitez avoir accès à l'info, vous devez interagir avec l'article en charge de l'info. Si vous êtes employé dans une entreprise, il y a de fortes chances que vous ayez eu une expérience de l'encapsulation. Pensez à un département des ressources humaines. Les membres du personnel des ressources humaines encapsulent (cachent) les données relatives aux employés. Ils déterminent comment ces données seront utilisées et manipulées. Toute demande de données sur les travailleurs ou toute demande de mise à jour des informations doit être acheminée par eux. En encapsulant les données, vous rendez les informations de votre système plus sûres et plus fiables. Vous êtes également en mesure de surveiller l'accès aux informations et les opérations qui sont effectuées sur celles-ci. Cela facilite la maintenance des programmes et simplifie le processus de débogage.

  2. Abstraction :

    3. L'abstraction consiste à utiliser des classes simples pour représenter la complexité. Fondamentalement, nous utilisons l'abstraction pour gérer la complexité en permettant à l'utilisateur de ne voir que les informations pertinentes et utiles. Un bon exemple pour expliquer cela est la conduite d'une voiture automatique. Lorsque vous avez une voiture automatique et que vous voulez vous rendre d'un point A à un point B, tout ce que vous devez faire est de lui donner la destination et de démarrer la voiture. Elle vous conduira ensuite à votre destination. Ce que vous n'avez pas besoin de savoir, c'est comment la voiture est fabriquée, comment elle prend et suit correctement les instructions, comment elle filtre les différentes options pour trouver le meilleur itinéraire, etc. Le même concept est appliqué lors de la construction d'applications OOP. Vous y parvenez en masquant les détails que l'utilisateur n'a pas besoin de voir. L'abstraction facilite les choses et vous permet de traiter vos projets en petites parties gérables.

  3. Heritages :

    4. L'héritage permet aux classes d'hériter des caractéristiques d'autres classes. Par exemple, vous pouvez classer tous les chats ensemble car ils ont certaines caractéristiques communes, comme le fait d'avoir quatre pattes. Leurs races les classent ensuite en sous-groupes ayant des attributs communs, comme la taille et la couleur. Dans la POO, vous utilisez l'héritage pour classer les objets de vos programmes en fonction de leurs caractéristiques et performances communes. Cela facilite le travail avec les objets et la programmation, car cela vous permet de mélanger des caractéristiques générales dans un objet parent et d'hériter de ces caractéristiques dans les objets enfants. Par exemple, vous allez définir un objet employé qui définit toutes les caractéristiques générales des employés de votre entreprise. Vous pourrez ensuite définir un objet "manager" qui hérite des caractéristiques de l'objet "employé", mais qui ajoute également des caractéristiques propres aux managers de votre entreprise. L'objet manager reflétera automatiquement tout changement dans l'implémentation de l'objet employé.

  4. Polymorphisme

    5. C'est le pouvoir de deux objets différents de répondre à un formulaire. Le programme déterminera quel usage est critique pour chaque exécution de la chose à partir de la classe parente, ce qui réduit la duplication du code. Il permet également à différents types d'objets d'interagir avec la même interface.

CONCLUSION :

Aujourd'hui, la plupart des langages permettent aux développeurs de mélanger les paradigmes de programmation. Cela est souvent dû au fait qu'ils seront utilisés pour diverses méthodes de programmation. Par exemple, prenez JavaScript - vous pouvez l'utiliser à la fois pour la programmation orientée objet et la programmation fonctionnelle. Lorsque vous codez du JS orienté objet, vous devez réfléchir soigneusement à la structure du programme et la planifier dès le début du codage. Vous pouvez le faire en voyant comment vous allez pouvoir décomposer les éléments nécessaires en classes simples et réutilisables qui seront utilisées pour créer des instances d'objets. Les développeurs travaillant avec la POO s'accordent généralement à dire qu'en général, son utilisation permet d'améliorer les structures de données et la réutilisabilité du code. Cela permet de gagner du temps sur le long terme.
